About

Type errors are a common stumbling block for people trying to learn functional programming languages like OCaml or Haskell. We are interested in learning what makes type errors difficult to diagnose and fix.

This survey is completely voluntary; you may stop at any time. We will not ask you for your name and will not record any identifying information. Data obtained from this survey will only be used to learn whether providing runtime traces helps diagnose and fix type errors.

If you choose to participate, you will be presented with examples of code written by students learning OCaml, and we will ask you to identify the error and fix it.

We recommend some familiarity with OCaml or functional programming to take this survey. We expect the survey to take roughly 30 minutes, but there is no time limit.

Instructions

You will be shown several snippets of code with type errors. For each example, we ask that you determine the cause of the error (i.e. which expression is at fault), explain why there is an error, and rewrite the code to fix the error.

Each example will come with either a simple error message, or an interactive trace of a failed execution. The interactive trace will start with just the input expression and the state at which the program got stuck (e.g. by trying to add an integer and a string). You can step forward or backward from a given expression by clicking on the expression and clicking the "Step forward" or "Step backward" button. You can also "Jump" (forward or backward) to the closest function call, skipping over intermediate steps.

In each expression, we will highlight the sub-expression where the next computation step will occur. For example, in 

x + y
we would highlight x because the next step is to lookup x in the environment. But in 
1 + 2
we would highlight the entire expression, as the next step is performing the addition. At times, the trace will need to disambiguate a variable by appending a | followed by a number, e.g. n may be renamed to n|1. Finally, you see the environment at any point in the trace by hovering over the corresponding expression.

We recommend taking a few minutes to familiarize yourself with a quick demo of the interactive trace before starting the survey.
